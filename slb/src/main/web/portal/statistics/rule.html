配置解释:

HTTP/2 采用异步连接、多路复用、请求/响应管线化等。 比HTTP1.0更简单,高效,强大
规则：
  • 首先要确认要开启HTTP2的域名所在SLB是否支持HTTP2。 当前外网的SLB默认开启HTTP2， 如不确认是否已经开启HTTP2.请联系 SLB Team
  • 确认当前VS协议是否为HTTPS。 对于HTTP的默认不支持HTTP2
  • 参数说明：
    • 包大小(Chunk Size):设置响应体被分割的最大块大小。过小的值会导致更高的负荷。太高的值由于排头阻塞对优化有害。
    • 请求体读取大小(Body Pre Read Size):设置每一个请求的缓冲区大小，请求体在开始处理前可以保存到缓冲区中
    • 空闲时长(Idle Time Out):设置非活跃的超时时间，过了这个时间后连接将被关闭。
    • 并发流大小(Cocurrent Streams):设置一个连接中同时传输HTTP2流的最大数。
    • 区间最大长度(Max Field Size):限制HPACK压缩请求头域的最大大小。限制应用于名字和值。注意如果使用哈夫曼编码，实际的解压的名字和值的字符串大小可能更大。对于大多数请求，默认的限制是足够的。
    • 最大请求头长度(Max Header Size):限制在HPACK解压后完整的请求头列表的最大大小。对于大多数请求来说，默认的限制是足够的。
    • 最大请求数(Max Requests):设置接收请求上限个数。
    • 接收缓存大小(Receive Buffer Size):设置接收缓冲区的大小。
    • 接收超时时间(Receive Timeout):为希望从客户端收到更多数据设置超时时间，超过这个时间之后连接将关闭
